## Security fixes (found by audit — were live in prod)

1. **Admin login page printed real admin credentials** (`+919999999999 / Test@1234`) at
   admin.washingbells.com → removed. **Rotate this password.**
2. **Unauthenticated `POST /admin/seed-admin`** → deleted (returns 404 in prod now).
3. **OTP dev bypass was implicit**: missing Twilio creds silently made `123456` a universal
   password for ANY phone. Now an explicit `OTP_DEV_BYPASS` env flag (default **fail closed**).
   Prod currently runs `OTP_DEV_BYPASS=true` on purpose (test phase, no Twilio yet).
4. **Payment verify was fail-open** without Razorpay keys → now only accepts dev-mock order ids.
5. **No OTP rate limiting** → 25s per-phone resend cooldown + 6/hour cap (`otp_requests`, 1h TTL).
6. **CORS `*` with credentials** → restricted to admin./www./washingbells.com + localhost dev.
7. Rider/store apps showed "DEV: Use OTP 123456" in production builds → gated behind `__DEV__`.
8. `TEST_CREDENTIALS.md`/`TEST_INFO.md` untracked from git (kept locally, gitignored);
   dead `.bak`/scratch files deleted.

## Feature work

**Weight-based orders (kg/grams):** services with `pricing_unit: kg` (prod has 3: Premium
Laundry, Wash & Fold, Wash and Iron) take fractional quantities. Admin New Order shows
kg+g inputs, price computed per kg; line items store `unit`; garment tags emit ONE tag per
kg line (it's a bag); invoice PDF prints `3.25 kg` / `₹79/kg`; orders drawer + bill editor
kg-aware; customer app shows `/kg` price labels and `x N kg` on order lines.
Verified on prod: order `WB-2026-O11T` — 3.25 kg @ ₹79 = ₹256.75 + ₹40 delivery, 1 tag.

**Coordinates optional:** rider-delivery orders need only the address text (riders navigate
by address string; map pin falls back to store coords). Admin UI lat/long inputs removed.
The standalone Add Customer modal never needed coords (verified).

**Sorting/categorization:** `/services` + `/admin/services` now return items sorted by
canonical category order then name (fixes YAML/$push insertion-order drift everywhere).
Customer home now renders the "Shop by Category" tiles (component existed but was never
mounted); category pages use the shared `matches()` so Men/Women/Kids include unisex items
(strict equality was hiding Shirt/Jeans/Kurta from "Men"); admin Services page has an item
search that auto-expands matching services.

**App auth/UX:** OTP-first on all 3 login screens; rider/store OTP screens gained
Resend-with-30s-countdown (had none); rider/store now persist refresh tokens; customer
OTP login registers push notifications (was password-login only); customer orders tab has
a real error + Retry state instead of a misleading "No orders yet" on network failure.

**Store-listing hygiene:** `RECORD_AUDIO` removed from rider/store (Play rejection risk);
expo-image-picker configured with camera/photo permission strings and `microphonePermission:
false` (also fixes iOS crash-on-camera without NSCameraUsageDescription); customer Android
permission duplicates removed.
